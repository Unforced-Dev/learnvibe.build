02 MCPs / Connectors

MCP stands for Model Context Protocol. In the Claude UI, you'll see these called Connectors or Integrations. They all mean the same thing: ways for Claude to reach beyond the conversation and interact with external services.

What MCPs Do

Without MCPs, Claude only knows what you tell it in the conversation. With MCPs, Claude can:

  • → Search the web for current information
  • → Access your Google Drive, Notion, or other services
  • → Connect to databases and APIs
  • → Read and write files on your computer (in Claude Desktop)

How to Think About It

Think of Claude as having "hands" that can reach out and touch things. By default, those hands are tied. MCPs untie them and let Claude interact with your world — your files, your data, your services.

Built-in Connectors in Claude

In the Claude web interface, you can enable connectors for:

Web Search
Let Claude search the internet for current information
Google Drive
Access your documents and files
Google Docs
Read and reference your documents
Notion
Connect your Notion workspace

Key Insight

The more context Claude has, the better it can help. MCPs are how you give Claude access to the context it needs — your real data, your real documents, your real world.

03 Skills

Skills are pre-packaged capabilities in Claude. Think of them as specialized modes or workflows that Claude can activate for specific tasks.

What Skills Offer

  • → Pre-built prompts and workflows for common tasks
  • → Optimized behavior for specific use cases
  • → Quick access to specialized capabilities

Skills are available in the Claude interface — look for them in the conversation or settings. They extend what Claude can do without you having to write complex prompts.

// try_it

Open Claude and explore the available Skills. Try activating one and see how it changes Claude's behavior for that task.

04 Claude Cowork

Claude Cowork is where Claude gets real power. It's not just conversation — it's Claude with the ability to create actual files on your computer.

"Cowork is basically like a chief of staff for $20 a month."

— From Week 1

What Cowork Can Do

  • → Create documents — Word docs, PowerPoints, spreadsheets
  • → Organize files — create folder structures, move things around
  • → Read your data — analyze CSVs, review documents
  • → Build systems — create templates, workflows, organized structures

Key Difference from Claude Desktop

In regular Claude Desktop, you can create artifacts — things that exist in the conversation. In Cowork, Claude creates real files that exist on your computer, that you can open in other programs, share with others, and build on over time.

05 Claude Code Introduction

Claude Code is Claude in your terminal. It can do everything Cowork can do — and more. It reads files, writes code, runs commands, and builds real software.

Cowork vs Claude Code

  • → Same capabilities — both can read/write files, organize content, build things
  • → Different interface — Cowork is visual, Claude Code is terminal-based
  • → Claude Code goes further — can run commands, install packages, deploy
  • → Platform availability — Claude Code works on Mac, Windows, and Linux

"You don't need to be a programmer to use Claude Code. You need to be able to articulate what you want — and that's the skill we've been practicing."

Demo: Same Workflow in Claude Code

We'll show the same meeting transcript workflow running in Claude Code — demonstrating that Windows users can do exactly what we just did in Cowork. The terminal looks different, but the result is the same.

It's Not Scary

The terminal might look intimidating, but Claude Code handles the complexity. You describe what you want in plain English, and Claude figures out the code. The same articulation skills you've been practicing apply here.

06 GitHub Basics

GitHub is where code lives on the internet. It's how developers store, share, and collaborate on software projects.

What GitHub Is

  • → A place to store your code (like Google Drive for software)
  • → Version control — keeps track of every change you make
  • → Collaboration — multiple people can work on the same project
  • → Hosting — can serve websites directly (like this course site)

Public vs Private

Public Repositories

  • • Anyone can see your code
  • • Great for open source projects
  • • Enables collaboration
  • • Free hosting for websites

Private Repositories

  • • Only you (and invited people) can see
  • • Good for client work or personal projects
  • • Protects proprietary code
  • • Free on GitHub (with limits)

Caution: What NOT to Commit

Never put these in a public repository:

  • • API keys — credentials for services
  • • Passwords — any login credentials
  • • .env files — environment variables with secrets
  • • Personal data — emails, phone numbers, etc.

If you accidentally commit secrets, consider them compromised and rotate them immediately.

Open Source

Open source means making your code public so others can use, learn from, and contribute to it. Benefits include:

  • → Transparency — people can see how things work
  • → Collaboration — others can help improve your project
  • → Learning — you learn from others' code, they learn from yours
  • → Community — building in public creates connection

Open source is powerful, but be thoughtful. Don't expose secrets. Don't share client data. When in doubt, start private and open up later.
